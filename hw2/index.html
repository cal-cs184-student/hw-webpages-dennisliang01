Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm evaluates a Bézier curve at a parameter t ∈ [0, 1] through repeated linear interpolation over a set of control points. At each step, every adjacent pair of points is blended together using the formula (1 − t) · pi + t · pi+1, producing a new, smaller set of points. This process repeats on each successive result until only a single point remains — that point is the location on the curve at parameter t. In my implementation, evaluateStep performs one such reduction: it iterates over adjacent pairs in the input vector and pushes each interpolated result into a new vector, which is returned. To fully evaluate the curve, this function is called recursively on its own output until the vector contains just one point, which for an n-th degree Bézier curve requires exactly n steps.

Part 2: Bezier surfaces with separable 1D de Casteljau

A Bézier patch extends de Casteljau's algorithm to two dimensions by applying it separably across two parameters, u and v. The control points form an n × n grid, and evaluation proceeds in two passes. In my implementation, evaluateStep performs a single linear interpolation pass identically to the 1D case, while evaluate1D calls it recursively until one point remains — fully evaluating a 1D Bézier curve at parameter t. In evaluate(u, v), each row of the control point grid is first evaluated at parameter u using evaluate1D, producing a single intermediate point per row. These intermediate points are then collected into a new vector and evaluated once more at parameter v, effectively sweeping the curve across the second axis. The result is the point on the surface at (u, v).

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To compute an area-weighted vertex normal, I traverse all non-boundary faces adjacent to the vertex using the halfedge data structure, cycling around by repeatedly following twin()->next() until returning to the starting halfedge. For each neighboring face, I retrieve the three vertex positions of that triangle and compute the cross product of two of its edge vectors — (p1 − p0) × (p2 − p0). This cross product naturally encodes both the face normal direction and its area as a magnitude, so larger triangles contribute more to the result. Each such vector is accumulated into a running sum, and once all neighbors have been visited the sum is normalized to produce a unit normal, giving a smooth, area-weighted approximation of the surface normal at that vertex.

Part 4: Edge flip

To implement the edge flip, I began by returning immediately if either neighboring face was a boundary. I then carefully collected all relevant halfedges, vertices, and faces before modifying anything, storing them in named local variables to avoid losing references mid-reassignment. With all elements captured, I used setNeighbors to reassign every halfedge's next, twin, vertex, edge, and face pointers in full, then updated the halfedge() pointers on all four vertices and both faces. Following the spec's recommendation, I reassigned every pointer unconditionally rather than trying to reason about which ones changed, which kept the logic straightforward and less error-prone.

The one bug I encountered was forgetting to update the face halfedge() pointers at the end. The flip appeared to work correctly on the first operation, but repeated flips produced corrupted geometry because a face could end up pointing to a halfedge that had been reassigned to the other face. The fix was simply adding f0->halfedge() = h0 and f1->halfedge() = h1 at the end of the function. This reinforced the importance of the spec's advice to treat every pointer as dirty and reset it explicitly, even when it seems unchanged.
